## 2026-06-22 - [Refactoring N+1 Query in SupplierOrderHandler]
**Learning:** Found an N+1 issue in `SupplierOrderHandler` (inside `api/barcode.go`) where multiple database inserts were performed in a loop (`tx.Exec`) for each generated barcode when ordering copies. Refactored this to use a single bulk insert operation via `tx.CopyFrom` combined with `pgx.CopyFromRows`.
**Action:** Always prefer `pgx.CopyFromRows` for batch database creation or insertion. This drastically reduces database round-trips when processing larger quantities (like bulk ordering of books).
## 2026-06-30 - [Refactoring N+1 Query in ImportStudentsHandler and ProcessOrder]
**Learning:** Found N+1 issues in `ImportStudentsHandler` (inside `api/import.go`) and `ProcessOrder` (inside `api/order_service.go`) where multiple database inserts were performed in loops (`tx.Exec`) when importing students and creating order positions. Refactored `ImportStudentsHandler` to use `UNNEST` for bulk upsert, and `ProcessOrder` to use `pgx.CopyFromRows`. Note that `pgx.CopyFromRows` only supports insert-only queries, and `UNNEST` supports upserts (ON CONFLICT DO UPDATE).
**Action:** Always prefer `pgx.CopyFromRows` for batch database insertion, and `UNNEST` with arrays for batch upsertion, to reduce database round-trips.
